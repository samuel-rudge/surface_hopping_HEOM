# SHEOM: Surface Hopping from Exact HEOM via TCL Master Equations

SHEOM implements a surface hopping framework derived from the **time-convolutionless master equation (TCLME)**, where the TCL generator is obtained from **exact Hierarchical Equations of Motion (HEOM)** propagation at fixed nuclear configuration.

The method couples classical nuclear (vibrational) dynamics with quantum dissipative electronic dynamics (molecule + metallic leads via HEOM), without heuristic hopping rules. Transition rates between electronic states are extracted directly from the exact reduced dynamics generated by HEOM.

This is research code intended for expert users familiar with HEOM, TCL projection techniques, and nonadiabatic molecular dynamics.

---

## Physical picture

### Electronic–bath dynamics (HEOM level)

For a given classical nuclear trajectory \(x(t)\), the full HEOM hierarchy evolves as:

rho_HEOM[x(t)] = ( rho_mol_el[x(t)], ADO_1[x(t)], ADO_2[x(t)], ... )

with equation of motion:

dot{rho_HEOM[x(t)]} = L_HEOM(x) rho_HEOM[x(t)]

The Liouvillian depends only on the instantaneous nuclear coordinate \(x\), not on the trajectory history.

---

### Time-local reduction (TCL construction)

Because full HEOM dynamics is available, the molecular electronic density matrix admits a time-local representation along a given trajectory:

dot{rho_mol_el[x(t)]} = L_TCLME[x(t)] rho_mol_el[x(t)]

The operator L_TCLME[x(t)] is obtained from the exact HEOM dynamics at fixed x(t) and defines an effective time-local generator including dissipative effects from the leads.

From this generator, transition rates between eigenstates of the isolated molecular Hamiltonian are extracted.

---

## Electronic surfaces and hopping mechanism

Electronic surfaces are defined as eigenstates of the isolated molecular Hamiltonian H_mol(x). These are constructed automatically by the code.

Electronic effects enter only through stochastic transitions between these surfaces. There is no mean-field backreaction and no phenomenological nonadiabatic coupling.

---

### Example: Holstein-type model

H_mol(x) = (epsilon + lambda * x) d† d + (Omega / 2)(x^2 + p^2)

This yields two electronic surfaces:

- neutral state
- charged state

Each surface defines a different classical force:

F_neutral = -Omega * x
F_charged = -Omega * x - lambda

---

## Surface hopping dynamics

For each trajectory:

1. propagate classical motion on current surface
2. evaluate L_TCLME[x(t)]
3. extract transition rates between electronic eigenstates
4. perform stochastic hop decision
5. update active surface

This defines a stochastic trajectory process fully derived from HEOM dynamics.

---

## Nuclear dynamics

Nuclei are propagated classically in phase space (x, p). Each trajectory evolves on a single electronic surface at a time.

- surface determines force
- hops change the active surface
- electronic influence enters only through surface-dependent potentials and stochastic transitions

Ensemble observables are obtained by averaging over independent trajectories.

---

## Code structure and workflow

The implementation is Python-driven at the top level, with performance-critical routines written in Fortran.

- Python:
  - trajectory propagation
  - HEOM–TCL coupling
  - surface hopping logic

- Fortran:
  - sparse HEOM Liouvillian construction
  - MKL/OpenMP accelerated propagation
  - x-dependent Liouvillian updates

The sparsity pattern of the HEOM Liouvillian is fixed. Only the values of nonzero elements depend on x and are updated during propagation.

---

## Inputs

### input_parameters.py

This file defines the full simulation setup:

- electronic structure (levels, energies, interactions)
- classical vibrational modes
- electron–vibration coupling
- HEOM hierarchy parameters
- bath parameters (chemical potentials, temperature, spectral decomposition)
- molecule–lead coupling
- surface hopping parameters
- number of trajectories

There is no spatial grid. All nuclear dynamics is trajectory-based.

---

### Trajectory control

- n_trajectories: number of independent trajectories
- nthreads_liouvillian: OpenMP threading for Liouvillian propagation

Initial conditions:

ic_type = "const_ic" | "wigner_ic" | "boltzmann_ic"

---

## Running the code

Serial execution:

python3 SHEOM_main.py

Parallel execution (joblib-based trajectory parallelization):

python3 SHEOM_main_parallelized.py

In the parallel version, all trajectories are propagated concurrently and written to shared output files.

---

## Outputs

All outputs are overwritten on each run.

### Ensemble-averaged outputs

- mol_pops.dat  
  Molecular electronic populations vs time

- active_surfaces_tracked.dat  
  Surface occupation probabilities vs time

---

### Trajectory-resolved outputs

- x_vec.dat  
  Nuclear coordinates  
  Shape: (n_time, n_trajectories)

- p_vec.dat  
  Nuclear momenta  
  Shape: (n_time, n_trajectories)

These are used for post-processing:
- fluctuations
- higher moments
- phase-space analysis

---

## Fortran compilation

The code requires compilation of Fortran extensions prior to execution.

A compilation script is provided:

./compile_f2py.sh

This builds:
- sparse HEOM utilities
- Liouvillian update routines
- MKL/OpenMP propagation kernels

## Scope and limitations

This is research code.

- no input validation
- no checkpointing or restart capability
- outputs are overwritten
- requires compatible Fortran + MKL + f2py environment

---

## Summary

SHEOM implements a surface hopping framework derived from exact HEOM dynamics via a time-convolutionless construction.

Electronic dynamics is treated quantum mechanically and nonperturbatively, nuclear dynamics is classical, and surface transitions are obtained from HEOM-derived time-local generators.

The method is intended for expert use in molecule–metal quantum dynamics.
